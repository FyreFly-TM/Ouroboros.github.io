Getting Started with Ouroboros

From zero to running code in 10 minutes.

üéØ Key Concept: Arrows Define Data Flow

In Ouroboros, arrows show where data goes. The most common is -> (forward arrow), which sends data left-to-right:

"Hello!" -> Print // Send to Print 5 -> Double -> Print // Chain through flows event -< Log, UI, Save // Fork to multiple handlers

Master the four core arrows (->, <-, <->, -<) and you've mastered the language. See Arrows Reference for details.

1. Installation

Ouroboros uses the Chimera compiler/interpreter.

  1. Download the latest .zip from the Downloads page
  2. Extract the zip file
  3. Create a new project folder for your scripts
  4. Copy chimera.exe into your project folder

VS Code Setup (Optional)

For syntax highlighting in Visual Studio Code:

  1. Open VS Code and go to Extensions (Ctrl+Shift+X)
  2. Click the three dots (‚ãØ) in the top right
  3. Select "Install from VSIX..."
  4. Choose the .vsix file from the downloaded package

2. Hello World

Create a file named hello.ouro in your project folder:

// hello.ouro "Hello, Ouroboros!" -> Print

Open a terminal in your project folder and run:

.\chimera.exe hello.ouro

Output:

Hello, Ouroboros!

The -> (forward arrow) sends the string to the Print function. This is the most common pattern in Ouroboros‚Äîdata flows from left to right.

3. Variables

Ouroboros has two declaration styles:

Mutable Variables (:=)

count := 0 // Declare and initialize count = 5 // Reassign later count += 1 // Compound assignment (now 6) count++ // Increment shorthand

Immutable Variables (=:)

PI =: 3.14159 // Cannot be reassigned MAX_HEALTH =: 100 // Use for constants

Data Types

// Numbers age := 25 price := 19.99 // Strings name := "Ouroboros" "Hello, " + name + "!" -> Print // Booleans active := true paused := false // Arrays items := [1, 2, 3, 4, 5] first := items[0] // Access by index items[1] = 99 // Modify by index

‚ö†Ô∏è Truthiness Note

In Ouroboros, 0 and "" (empty string) are truthy. Only null and false are falsy.

4. Flows (Functions)

Flows are Ouroboros functions. Define them with the Flow keyword:

Basic Flow

Flow Greet(name) "Hello, " + name + "!" -> Print return true // Call it Greet("World") Greet("Alice")

Flow with Return Value

Flow Add(a, b) sum := a + b return sum // Capture the return value result := Add(10, 5) "Sum: " + result -> Print // Sum: 15.00

Multiple Return Values

Flow Divide(a, b) quotient := Floor(a / b) remainder := a % b return quotient, remainder // Capture both with pull arrow q, r <- Divide(17, 5) "17 √∑ 5 = " + q + " remainder " + r -> Print

üí° Flow Tip

Flow bodies are indented with 4 spaces (not tabs). This is significant in Ouroboros!

5. Basic Arrows

Arrows are the heart of Ouroboros. Here are the four you'll use most:

Forward Arrow (->)

Push data to a destination:

"Hello!" -> Print // Output to console 42 -> ProcessNumber // Pass to a flow data -> Step1 -> Step2 -> Print // Chain operations

Pull Arrow (<-)

Capture return values:

result <- Calculate(10, 5) // Capture return x, y <- GetPosition() // Multiple returns

Sync Arrow (<->)

Keep two variables in sync:

model <-> view // Changes to either update both model = 100 // view is now also 100 view = 50 // model is now also 50

Fork Arrow (-<)

Send to multiple destinations at once:

event -< Logger, UI, Analytics // All three receive the event

For the complete arrow reference, see the Arrows documentation.

6. Control Flow

If / Elseif / Else

score := 85 if score ‚â• 90 "Grade: A" -> Print elseif score ‚â• 80 "Grade: B" -> Print elseif score ‚â• 70 "Grade: C" -> Print else "Grade: F" -> Print

‚å®Ô∏è Unicode Operators

Use ‚â• (Alt+242) and ‚â§ (Alt+243) for comparisons. You can also use >= and <=.

While Loop

i := 0 while i < 5 "Count: " + i -> Print i++

For Loop

Uses comma-separated clauses (not semicolons):

// init, condition, increment for i = 0, i < 5, i++ "i = " + i -> Print

Early Exit with kill

for i = 0, i < 100, i++ if i == 5 kill // Exit the loop i -> Print
