Getting Started with Ouroboros

From zero to running code in 10 minutes.

1. Installation

Ouroboros uses the Chimera compiler/interpreter.

  1. Download the latest .zip from the Downloads page
  2. Extract the zip file
  3. Create a new project folder for your scripts
  4. Copy chimera.exe into your project folder

VS Code Setup (Optional)

For syntax highlighting in Visual Studio Code:

  1. Open VS Code and go to Extensions (Ctrl+Shift+X)
  2. Click the three dots (⋯) in the top right
  3. Select "Install from VSIX..."
  4. Choose the .vsix file from the downloaded package

2. Hello World

Create a file named hello.ouro in your project folder:

// hello.ouro "Hello, Ouroboros!" -> Print

Open a terminal in your project folder and run:

// Run a specific script .\chimera.exe hello.ouro // Or run the entire project .\chimera.exe

Output:

Hello, Ouroboros!

The -> (forward arrow) sends the string to the Print function.

3. Variables

Ouroboros has two declaration styles:

Mutable Variables (:=)

count := 0 // Declare and initialize count = 5 // Reassign count += 1 // Compound assignment (now 6)

Immutable Variables (=:)

pi =: 3.14159 // Cannot be reassigned later

Data Types

// Numbers age := 25 price := 19.99 negative := -42 // Strings name := "Ouroboros" // Booleans active := true paused := false // Arrays numbers := [1, 2, 3, 4, 5] mixed := [42, "text", true] // Access array elements first := numbers[0]

4. Flows (Functions)

Flows are the building blocks of Ouroboros—similar to functions in other languages.

Basic Flow

Flow Greet -> name "Hello, " -> Print name -> Print return true // Call the flow "World" -> Greet

Flow with Return Value

Flow Add -> a, b result := a + b return result // Capture the return value with a pull arrow sum <- Add -> 10, 5 sum -> Print // Outputs: 15

Multi-Return Flow

Flow Divide -> a, b quotient := a / b remainder := a % b return quotient, remainder // Capture multiple return values q, r <- Divide -> 17, 5 "Quotient: " -> Print q -> Print // 3 "Remainder: " -> Print r -> Print // 2

5. Basic Arrows

Arrows are the core of Ouroboros. Here are the essentials:

ArrowNamePurpose
-> Forward Push data to a flow or expression
<- Pull Capture return values or bind reactively
<-> Sync Bidirectional binding
-< Fork Send to multiple destinations

Forward Arrow Examples

// Send value to Print 42 -> Print // Chain multiple flows data -> Process -> Validate -> Print // Pass multiple arguments 10, 5 -> Add -> Print

Pull Arrow Examples

// Capture return value result <- Calculate -> input // Reactive binding (updates when dependencies change) area <- width * height

Sync Arrow Example

// Both variables stay in sync left := 10 right := 20 left <-> right right = 50 // left is now also 50

Fork Arrow Example

// Send to multiple flows at once event -< LogEvent, UpdateUI, NotifyUser

6. Control Flow

If / Elseif / Else

Blocks are defined by indentation (4 spaces). Note: use ≥ and ≤ for comparisons.

score := 85 if score ≥ 90 grade := "A" "Excellent!" -> Print elseif score ≥ 80 grade := "B" "Good job!" -> Print elseif score ≥ 70 grade := "C" "Passing" -> Print else grade := "F" "Needs improvement" -> Print

While Loop

counter := 0 while counter < 5 counter -> Print counter += 1 // Outputs: 0 1 2 3 4

Comparison Operators

OperatorMeaning
==Equal
!=Not equal
>Greater than
<Less than
≥Greater than or equal
≤Less than or equal

Logical Operators

// AND if active & ready "Go!" -> Print // OR if paused | stopped "Not running" -> Print // NOT if !locked "Accessible" -> Print
