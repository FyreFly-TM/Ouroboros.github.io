Getting Started with Ouroboros

From zero to running code in 10 minutes.

1. Installation

Ouroboros uses the Chimera compiler/interpreter.

  1. Download the latest .zip from the Downloads page
  2. Extract the zip file
  3. Create a new project folder for your scripts
  4. Copy chimera.exe into your project folder

VS Code Setup (Optional)

For syntax highlighting in Visual Studio Code:

  1. Open VS Code and go to Extensions (Ctrl+Shift+X)
  2. Click the three dots (⋯) in the top right
  3. Select "Install from VSIX..."
  4. Choose the .vsix file from the downloaded package

2. Hello World

Create a file named hello.ouro in your project folder:

// hello.ouro "Hello, Ouroboros!" -> Print

Open a terminal in your project folder and run:

// Run a specific script .\chimera.exe hello.ouro // Or run the entire project .\chimera.exe

Output:

Hello, Ouroboros!

The -> (forward arrow) sends the string to the Print function.

3. Variables

Ouroboros has two declaration styles:

Mutable Variables (:=)

count := 0 // Declare and initialize count = 5 // Reassign count += 1 // Compound assignment (now 6)

Immutable Variables (=:)

pi =: 3.14159 // Cannot be reassigned later

Data Types

// COMPLETE EXAMPLE: Data Types // Save as: data_types.ouro "=== Data Types Demo ===" -> Print // Numbers age := 25 price := 19.99 negative := -42 "Numbers:" -> Print "age = " + age -> Print "price = " + price -> Print "negative = " + negative -> Print // Strings name := "Ouroboros" "" -> Print "String: name = " + name -> Print // Booleans active := true paused := false "" -> Print "Booleans:" -> Print "active = " + active -> Print "paused = " + paused -> Print // Arrays numbers := [1, 2, 3, 4, 5] "" -> Print "Array: numbers = [1, 2, 3, 4, 5]" -> Print // Access array elements first := numbers[0] "first element = " + first -> Print // Modify array elements numbers[1] = 99 "After numbers[1] = 99: " + numbers[1] -> Print // Expected Output: // === Data Types Demo === // Numbers: // age = 25.00 // price = 19.99 // negative = -42.00 // // String: name = Ouroboros // // Booleans: // active = true // paused = false // // Array: numbers = [1, 2, 3, 4, 5] // first element = 1.00 // After numbers[1] = 99: 99.00

4. Flows (Functions)

Flows are the building blocks of Ouroboros—similar to functions in other languages.

Basic Flow

// COMPLETE EXAMPLE: Basic Flow // Save as: greet_flow.ouro "=== Basic Flow Demo ===" -> Print Flow Greet(name) "Hello, " + name + "!" -> Print return true // Call the flow Greet("World") Greet("Ouroboros") Greet("User") // Expected Output: // === Basic Flow Demo === // Hello, World! // Hello, Ouroboros! // Hello, User!

Flow with Return Value

// COMPLETE EXAMPLE: Flow with Return // Save as: add_flow.ouro "=== Flow with Return Demo ===" -> Print Flow Add(a, b) result := a + b return result // Capture the return value sum := Add(10, 5) "10 + 5 = " + sum -> Print sum := Add(100, 200) "100 + 200 = " + sum -> Print sum := Add(7, 3) "7 + 3 = " + sum -> Print // Expected Output: // === Flow with Return Demo === // 10 + 5 = 15.00 // 100 + 200 = 300.00 // 7 + 3 = 10.00

Multi-Return Flow

// COMPLETE EXAMPLE: Multi-Return Flow // Save as: divide_flow.ouro "=== Multi-Return Flow Demo ===" -> Print Flow Divide(a, b) quotient := a / b remainder := a % b return quotient, remainder // Capture multiple return values "Dividing 17 by 5:" -> Print q, r <- Divide(17, 5) "Quotient: " + q -> Print "Remainder: " + r -> Print "" -> Print "Dividing 100 by 7:" -> Print q, r <- Divide(100, 7) "Quotient: " + q -> Print "Remainder: " + r -> Print // Expected Output: // === Multi-Return Flow Demo === // Dividing 17 by 5: // Quotient: 3.40 // Remainder: 2.00 // // Dividing 100 by 7: // Quotient: 14.29 // Remainder: 2.00

Implicit Return Binding

When a Flow returns named values, those variable names become automatically accessible in the calling scope without explicit assignment:

// COMPLETE EXAMPLE: Implicit Return Binding // Save as: implicit_binding.ouro "=== Implicit Return Binding Demo ===" -> Print Flow Calculate(x, y) sum := x + y product := x * y return sum, product // Just call the flow - no assignment needed! "Calculating with 10 and 5:" -> Print Calculate(10, 5) // Now sum and product are available directly "sum = " + sum -> Print "product = " + product -> Print "" -> Print "Calculating with 7 and 3:" -> Print Calculate(7, 3) "sum = " + sum -> Print "product = " + product -> Print // Expected Output: // === Implicit Return Binding Demo === // Calculating with 10 and 5: // sum = 15.00 // product = 50.00 // // Calculating with 7 and 3: // sum = 10.00 // product = 21.00

This is a key feature of Ouroboros—return values are bound by name, not by position. Learn more in the Flows reference.

5. Basic Arrows

Arrows are the core of Ouroboros. Here are the essentials:

ArrowNamePurpose
-> Forward Push data to a flow or expression
<- Pull Capture return values or bind reactively
<-> Sync Bidirectional binding
-< Fork Send to multiple destinations

Forward Arrow Examples

// COMPLETE EXAMPLE: Forward Arrow // Save as: forward_arrow.ouro "=== Forward Arrow Demo ===" -> Print Flow Double(n) result := n * 2 return result Flow Square(n) result := n * n return result // Send value to Print "Sending 42 to Print:" -> Print 42 -> Print // Call flows with arrow "" -> Print "Double(10):" -> Print Double(10) result -> Print "" -> Print "Square(5):" -> Print Square(5) result -> Print // Expected Output: // === Forward Arrow Demo === // Sending 42 to Print: // 42.00 // // Double(10): // 20.00 // // Square(5): // 25.00

Pull Arrow Examples

// COMPLETE EXAMPLE: Pull Arrow // Save as: pull_arrow.ouro "=== Pull Arrow Demo ===" -> Print Flow Calculate(a, b) sum := a + b return sum // Capture return value with pull arrow result <- Calculate(10, 5) "Calculate(10, 5) returned: " + result -> Print // Reactive binding demonstration width := 10 height := 5 area := width * height "" -> Print "Initial:" -> Print "width = " + width -> Print "height = " + height -> Print "area = " + area -> Print // Update and recalculate width = 20 area = width * height "" -> Print "After width = 20:" -> Print "area = " + area -> Print // Expected Output: // === Pull Arrow Demo === // Calculate(10, 5) returned: 15.00 // // Initial: // width = 10.00 // height = 5.00 // area = 50.00 // // After width = 20: // area = 100.00

Sync Arrow Example

// COMPLETE EXAMPLE: Sync Arrow // Save as: sync_arrow.ouro "=== Sync Arrow Demo ===" -> Print // Both variables stay in sync left := 10 right := 20 "Initial values:" -> Print "left = " + left -> Print "right = " + right -> Print left <-> right "" -> Print "After sync binding (no immediate change):" -> Print "left = " + left -> Print "right = " + right -> Print "" -> Print "Setting right = 50..." -> Print right = 50 "left = " + left -> Print "right = " + right -> Print // Expected Output: // === Sync Arrow Demo === // Initial values: // left = 10.00 // right = 20.00 // // After sync binding (no immediate change): // left = 10.00 // right = 20.00 // // Setting right = 50... // left = 50.00 // right = 50.00

Fork Arrow Example

// COMPLETE EXAMPLE: Fork Arrow // Save as: fork_arrow.ouro "=== Fork Arrow Demo ===" -> Print Flow LogEvent(event) "[LOG] " + event -> Print return true Flow UpdateUI(event) "[UI] " + event -> Print return true Flow NotifyUser(event) "[NOTIFY] " + event -> Print return true // Send to multiple flows at once "Forking 'user_login' to 3 handlers:" -> Print "" -> Print event := "user_login" event -< LogEvent, UpdateUI, NotifyUser // Expected Output: // === Fork Arrow Demo === // Forking 'user_login' to 3 handlers: // // [LOG] user_login // [UI] user_login // [NOTIFY] user_login

6. Control Flow

If / Elseif / Else

Blocks are defined by indentation (4 spaces). Use ≥ and ≤ for comparisons (not ASCII >= or <=).

// COMPLETE EXAMPLE: If/Elseif/Else // Save as: grading.ouro "=== Grading System ===" -> Print Flow GetGrade(score) grade := "" if (score ≥ 90) grade = "A" "Excellent!" -> Print elseif (score ≥ 80) grade = "B" "Good job!" -> Print elseif (score ≥ 70) grade = "C" "Passing" -> Print else grade = "F" "Needs improvement" -> Print return grade "Score 85:" -> Print GetGrade(85) "Grade: " + grade -> Print "" -> Print "Score 95:" -> Print GetGrade(95) "Grade: " + grade -> Print // Expected Output: // === Grading System === // Score 85: // Good job! // Grade: B // // Score 95: // Excellent! // Grade: A

While Loop

// COMPLETE EXAMPLE: While Loop // Save as: counter.ouro "=== While Loop Demo ===" -> Print counter := 0 "Counting from 0 to 4:" -> Print while (counter < 5) "counter = " + counter -> Print counter = counter + 1 "" -> Print "Loop complete!" -> Print "Final counter = " + counter -> Print // Expected Output: // === While Loop Demo === // Counting from 0 to 4: // counter = 0.00 // counter = 1.00 // counter = 2.00 // counter = 3.00 // counter = 4.00 // // Loop complete! // Final counter = 5.00

For Loop

A cleaner syntax with initialization, condition, and increment in one line:

// COMPLETE EXAMPLE: For Loop // Save as: for_loop.ouro "=== For Loop Demo ===" -> Print "Counting 0 to 4:" -> Print for i = 0, i < 5, i++ "i = " + i -> Print "" -> Print "Sum of 1 to 5:" -> Print sum := 0 for n = 1, n ≤ 5, n++ sum = sum + n "Total: " + sum -> Print // Expected Output: // === For Loop Demo === // Counting 0 to 4: // i = 0.00 // i = 1.00 // i = 2.00 // i = 3.00 // i = 4.00 // // Sum of 1 to 5: // Total: 15.00

Comparison Operators

Important: Use Unicode ≥ and ≤ for greater/less-or-equal comparisons.

OperatorMeaning
==Equal
!=Not equal
>Greater than
<Less than
≥Greater than or equal
≤Less than or equal

Logical Operators

// COMPLETE EXAMPLE: Logical Operators // Save as: logic.ouro "=== Logical Operators Demo ===" -> Print active := true ready := true paused := false locked := false "Variables:" -> Print "active = " + active -> Print "ready = " + ready -> Print "paused = " + paused -> Print "locked = " + locked -> Print "" -> Print // AND if (active & ready) "AND: active & ready = Go!" -> Print // OR if (paused | locked) "OR: paused | locked = Blocked" -> Print else "OR: Neither paused nor locked" -> Print // NOT if (!locked) "NOT: !locked = Accessible" -> Print // Expected Output: // === Logical Operators Demo === // Variables: // active = true // ready = true // paused = false // locked = false // // AND: active & ready = Go! // OR: Neither paused nor locked // NOT: !locked = Accessible
