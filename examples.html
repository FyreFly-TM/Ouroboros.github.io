Ouroboros Examples

From simple scripts to complex simulations—working code you can run today.

Beginner Examples

Hello World

The simplest Ouroboros program.

"Hello, World!" -> Print

Variables and Math

Basic variable declaration and arithmetic.

// Declare variables x := 10 y := 5 // Arithmetic operations sum := x + y diff := x - y product := x * y quotient := x / y remainder := x % y // Print results "Sum: " -> Print sum -> Print "Product: " -> Print product -> Print

Simple Flow

Define and call a flow (function).

Flow Double -> n result := n * 2 return result // Call the flow and capture result answer <- Double -> 21 answer -> Print // Outputs: 42

Greeting Flow

A flow that prints a personalized greeting.

Flow Greet -> name "Hello, " -> Print name -> Print "!" -> Print return true "Alice" -> Greet "Bob" -> Greet

Counter Loop

A simple while loop counting from 1 to 5.

i := 1 while i ≤ 5 i -> Print i += 1 "Done!" -> Print

Conditional Logic

Using if/elseif/else to grade a score.

score := 78 if score ≥ 90 "Grade: A" -> Print elseif score ≥ 80 "Grade: B" -> Print elseif score ≥ 70 "Grade: C" -> Print elseif score ≥ 60 "Grade: D" -> Print else "Grade: F" -> Print

Intermediate Examples

Multi-Return Flow

A flow that returns multiple values at once.

Flow ComputeStats -> a, b sum := a + b product := a * b average := sum / 2 return sum, product, average // Capture all three return values s, p, avg <- ComputeStats -> 10, 20 "Sum: " -> Print s -> Print "Product: " -> Print p -> Print "Average: " -> Print avg -> Print

Calculator

A modular calculator using multiple flows.

Flow Add -> a, b return a + b Flow Subtract -> a, b return a - b Flow Multiply -> a, b return a * b Flow Divide -> a, b return a / b // Use the calculator result1 <- Add -> 15, 25 result2 <- Multiply -> result1, 2 result2 -> Print // 80

Flow Chaining

Chain multiple flows together in a pipeline.

Flow AddOne -> n return n + 1 Flow Double -> n return n * 2 Flow Square -> n return n * n // Chain: 5 -> AddOne (6) -> Double (12) -> Square (144) 5 -> AddOne -> Double -> Square -> Print

Array Processing

Working with arrays and indexing.

// Create arrays numbers := [10, 20, 30, 40, 50] names := ["Alice", "Bob", "Charlie"] // Access elements first := numbers[0] last := numbers[4] "First: " -> Print first -> Print "Last: " -> Print last -> Print // Create array from flow Flow MakeTriple -> a, b, c return [a, b, c] triple <- MakeTriple -> 1, 2, 3 triple -> Print

Reactive Bindings

Create variables that update automatically.

width := 10 height := 5 // area automatically updates when width or height change area <- width * height perimeter <- 2 * (width + height) "Initial area: " -> Print area -> Print // 50 // Change a dimension width = 20 "New area: " -> Print area -> Print // 100 (auto-updated)

Bidirectional Sync

Keep two variables perfectly synchronized.

model_value := 100 view_value := 0 // Establish sync model_value <-> view_value "After sync - view: " -> Print view_value -> Print // 100 // Change either side view_value = 250 "Model after view change: " -> Print model_value -> Print // 250 model_value = 999 "View after model change: " -> Print view_value -> Print // 999

Fork Broadcasting

Send an event to multiple handlers simultaneously.

Flow LogEvent -> event "LOG: Event received - " -> Print event -> Print return true Flow UpdateUI -> event "UI: Refreshing display for - " -> Print event -> Print return true Flow NotifyUser -> event "NOTIFY: Alert sent for - " -> Print event -> Print return true // Fork: send to all three handlers at once "user_login" -< LogEvent, UpdateUI, NotifyUser

Advanced Examples

Conditional Gates

Route data based on conditions using gate syntax.

Flow ProcessHigh -> val "Processing HIGH value: " -> Print val -> Print return true Flow ProcessLow -> val "Processing LOW value: " -> Print val -> Print return true value := 75 // Gate: only executes if condition is true value -[value > 50]-> ProcessHigh value -[value ≤ 50]-> ProcessLow

Transform Gates

Transform data inline before passing to a flow.

Flow Receiver -> data "Received: " -> Print data -> Print return true input := 10 // Transform: if condition is true, pass transformed value input -[input > 5 ? input * 2]-> Receiver // Receives 20 input -[input > 5 ? "HIGH"]-> Receiver // Receives "HIGH"

Continuous Funnels

React to value changes automatically.

sensor_temp := 22.0 sensor_humidity := 45.0 Flow ClimateMonitor -> val "Climate sensor triggered: " -> Print val -> Print return true // Continuous funnel: triggers on any change sensor_temp ▷- ClimateMonitor sensor_humidity ▷- ClimateMonitor // Changing values triggers the monitor "Simulating temperature change..." -> Print sensor_temp = 28.5

Fuzzy Logic Arrow

Soft influence—suggest rather than force values.

target_temp := 22.0 Flow ThermalController -> t if t > 25.0 "Fuzzy: Cooling suggested" -> Print elseif t < 18.0 "Fuzzy: Heating suggested" -> Print else "Fuzzy: Temperature nominal" -> Print return true // Fuzzy arrow: soft influence target_temp ~> ThermalController

Solver Arrow

Constraint-based solving for pathfinding or optimization.

Flow PathSolver -> start "SOLVER: Calculating optimal path from " -> Print start -> Print "SOLVER: Path found via A* algorithm" -> Print return "optimal_path" current_position := "Sector_A" // Solver arrow: find solution current_position => PathSolver

Ownership Transfer

Transfer data ownership—source becomes invalid.

Flow DepotReceiver -> cargo "DEPOT: Received cargo:" -> Print cargo -> Print return true drone_cargo := ["Titanium", "Gold", "Uranium"] "Drone cargo before transfer:" -> Print drone_cargo -> Print // Transfer ownership drone_cargo -|-> DepotReceiver // drone_cargo is now conceptually invalid

Sever Connection

Explicitly break a connection or trigger disconnect.

Flow DisconnectHandler -> id "Disconnecting: " -> Print id -> Print "Connection severed." -> Print return true session_id := "Session_12345" // Sever arrow: break connection session_id -||-> DisconnectHandler

Recursive Flow

A flow that calls itself for recursive algorithms.

Flow Factorial -> n if n ≤ 1 return 1 prev <- Factorial -> (n - 1) return n * prev result <- Factorial -> 5 "5! = " -> Print result -> Print // 120

Complex Chain

Combine multiple arrow types in a single pipeline.

Flow Decrypt -> data "Decrypting..." -> Print return "decrypted_data" Flow Analyze -> data "Analyzing intent..." -> Print return data Flow Solve -> data "Solving geometry..." -> Print return "solved" Flow LogResult -> result "LOGGED: " -> Print result -> Print return true Flow AlertTeam -> result "ALERT SENT" -> Print return true // The Omega Chain: Gate -> Flow -> Fuzzy -> Solver -> Fork radiation := 3.0 artifact := "alien_signal" artifact -[radiation < 10]-> Decrypt ~> Analyze => Solve -< LogResult, AlertTeam

Common Patterns

Event System

A simple event broadcasting pattern.

// Event handlers Flow OnPlayerJoin -> player "Player joined: " -> Print player -> Print return true Flow OnPlayerLeave -> player "Player left: " -> Print player -> Print return true Flow LogActivity -> player "Activity logged for: " -> Print player -> Print return true // Broadcast player events "Alice" -< OnPlayerJoin, LogActivity "Bob" -< OnPlayerLeave, LogActivity

State Machine

Using conditional gates as a state machine.

state := "idle" Flow HandleIdle -> s "State: IDLE -> Starting" -> Print return "running" Flow HandleRunning -> s "State: RUNNING -> Processing" -> Print return "complete" Flow HandleComplete -> s "State: COMPLETE -> Done!" -> Print return "idle" // Process based on current state state -[state == "idle"]-> HandleIdle state -[state == "running"]-> HandleRunning state -[state == "complete"]-> HandleComplete

Pipeline Processing

Data transformation pipeline.

Flow Validate -> data "Validating..." -> Print return data Flow Transform -> data "Transforming..." -> Print return data Flow Store -> data "Storing..." -> Print return true // Process data through pipeline raw_data := "user_input" raw_data -> Validate -> Transform -> Store

Sensor Fusion

Multiple sensors feeding into a single processor.

Flow SensorProcessor -> reading "Sensor reading: " -> Print reading -> Print return true // Multiple sensors continuously feed the processor temperature := 22.5 pressure := 1013.25 humidity := 65.0 temperature ▷- SensorProcessor pressure ▷- SensorProcessor humidity ▷- SensorProcessor // Any change triggers processing temperature = 23.0
