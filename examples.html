Sequential vs Parallel Execution

Use comma for sequential and colon for parallel execution.

// COMPLETE EXAMPLE: Sequential vs Parallel Execution // Save as: seq_parallel.ouro "=== Sequential vs Parallel Execution ===" -> Print Flow Log(msg) "LOG: " + msg -> Print return msg Flow FetchData(source) "Fetching from " + source + "..." -> Print return "Data_" + source "" -> Print "--- Sequential Execution (comma) ---" -> Print // Sequential: steps happen in order, one after another Log("Step 1"), Log("Step 2"), Log("Step 3") "" -> Print "--- Parallel Execution (colon) ---" -> Print // Parallel: all three start at once FetchData("SourceA") : FetchData("SourceB") : FetchData("SourceC") "" -> Print "--- Mixed Execution ---" -> Print // First two parallel, then sequential FetchData("Alpha") : FetchData("Beta"), Log("Both fetches complete") // Expected Output: // === Sequential vs Parallel Execution === // // --- Sequential Execution (comma) --- // LOG: Step 1 // LOG: Step 2 // LOG: Step 3 // // --- Parallel Execution (colon) --- // Fetching from SourceA... // Fetching from SourceB... // Fetching from SourceC... // // --- Mixed Execution --- // Fetching from Alpha... // Fetching from Beta... // LOG: Both fetches complete

Bidirectional Sync

Keep two variables perfectly synchronized.

// COMPLETE EXAMPLE: Bidirectional Sync // Save as: bidirectional_sync.ouro "=== Bidirectional Sync ===" -> Print // Two separate variables model_value := 100 view_value := 0 "--- Before Sync ---" -> Print "Model: " + model_value -> Print "View: " + view_value -> Print "" -> Print "--- Establishing Sync (<->) ---" -> Print // The sync arrow links them bidirectionally model_value <-> view_value "Model: " + model_value -> Print "View: " + view_value -> Print "" -> Print "--- Change View to 250 ---" -> Print view_value = 250 "Model (auto-synced): " + model_value -> Print "View: " + view_value -> Print "" -> Print "--- Change Model to 999 ---" -> Print model_value = 999 "Model: " + model_value -> Print "View (auto-synced): " + view_value -> Print "" -> Print "--- Change View to 42 ---" -> Print view_value = 42 "Model (auto-synced): " + model_value -> Print "View: " + view_value -> Print // Expected Output: // === Bidirectional Sync === // --- Before Sync --- // Model: 100.00 // View: 0.00 // // --- Establishing Sync (<->) --- // Model: 100.00 // View: 100.00 // // --- Change View to 250 --- // Model (auto-synced): 250.00 // View: 250.00 // // --- Change Model to 999 --- // Model: 999.00 // View (auto-synced): 999.00 // // --- Change View to 42 --- // Model (auto-synced): 42.00 // View: 42.00

Conditional Gates

Route data based on conditions using gate syntax.

// COMPLETE EXAMPLE: Conditional Gates // Save as: conditional_gates.ouro "=== Conditional Gates ===" -> Print Flow ProcessHigh(val) " HIGH: Processing value " + val + " (above threshold)" -> Print return val * 2 Flow ProcessLow(val) " LOW: Processing value " + val + " (below threshold)" -> Print return val / 2 Flow ProcessZero(val) " ZERO: Value is exactly zero" -> Print return 0 // Test multiple values value1 := 75 value2 := 30 value3 := 0 "--- Testing value: 75 ---" -> Print // Gate: only executes if condition is true value1 -[value1 > 50]-> ProcessHigh value1 -[value1 <= 50]-> ProcessLow "" -> Print "--- Testing value: 30 ---" -> Print value2 -[value2 > 50]-> ProcessHigh value2 -[value2 <= 50]-> ProcessLow "" -> Print "--- Testing value: 0 ---" -> Print value3 -[value3 == 0]-> ProcessZero value3 -[value3 > 50]-> ProcessHigh value3 -[value3 < 50]-> ProcessLow // Expected Output: // === Conditional Gates === // --- Testing value: 75 --- // HIGH: Processing value 75.00 (above threshold) // // --- Testing value: 30 --- // LOW: Processing value 30.00 (below threshold) // // --- Testing value: 0 --- // ZERO: Value is exactly zero

Fuzzy Logic Arrow

Soft influence—suggest rather than force values.

// COMPLETE EXAMPLE: Fuzzy Logic Arrow // Save as: fuzzy_logic.ouro "=== Fuzzy Logic Arrow ===" -> Print Flow ThermalController(t) " Controller received temp: " + t -> Print if t > 25.0 " Recommendation: Cooling suggested" -> Print elseif t < 18.0 " Recommendation: Heating suggested" -> Print else " Recommendation: Temperature nominal" -> Print return true Flow ComfortAnalyzer(t) " Analyzing comfort level..." -> Print comfort := 100 - ((t - 22) * (t - 22)) " Comfort score: " + comfort -> Print return comfort "--- Testing warm temperature (28.0) ---" -> Print target_temp := 28.0 // Fuzzy arrow: soft influence, not a hard assignment target_temp ~> ThermalController "" -> Print "--- Testing cold temperature (15.0) ---" -> Print target_temp = 15.0 target_temp ~> ThermalController "" -> Print "--- Testing ideal temperature (22.0) ---" -> Print target_temp = 22.0 target_temp ~> ThermalController target_temp ~> ComfortAnalyzer // Expected Output: // === Fuzzy Logic Arrow === // --- Testing warm temperature (28.0) --- // Controller received temp: 28.00 // Recommendation: Cooling suggested // // --- Testing cold temperature (15.0) --- // Controller received temp: 15.00 // Recommendation: Heating suggested // // --- Testing ideal temperature (22.0) --- // Controller received temp: 22.00 // Recommendation: Temperature nominal // Analyzing comfort level... // Comfort score: 100.00

Fuzzy Pull & Sync (<~, <~>)

// COMPLETE EXAMPLE: Fuzzy Pull and Sync // Save as: fuzzy_pull_sync.ouro "=== Fuzzy Pull & Sync ===" -> Print "--- Fuzzy Pull (<~) ---" -> Print // Fuzzy Pull: AI perceives player with uncertainty player_visibility := 0.8 enemy_alert := 0.0 "Player visibility: " + player_visibility -> Print "Enemy alert before fuzzy pull: " + enemy_alert -> Print // Soft pull - value may include uncertainty/noise enemy_alert <~ player_visibility "Enemy alert after fuzzy pull: " + enemy_alert -> Print "" -> Print "--- Fuzzy Sync (<~>) ---" -> Print // Fuzzy Sync: elastic bidirectional influence market_supply := 500.0 market_demand := 600.0 "Initial supply: " + market_supply -> Print "Initial demand: " + market_demand -> Print // Changes propagate with elasticity (soft sync) market_supply <~> market_demand "After fuzzy sync:" -> Print "Market supply: " + market_supply -> Print "Market demand: " + market_demand -> Print "" -> Print "--- Changing synced values ---" -> Print market_supply = 700.0 "Supply changed to 700" -> Print "Market supply: " + market_supply -> Print "Market demand: " + market_demand -> Print // Expected Output: // === Fuzzy Pull & Sync === // --- Fuzzy Pull (<~) --- // Player visibility: 0.80 // Enemy alert before fuzzy pull: 0.00 // Enemy alert after fuzzy pull: 0.80 // // --- Fuzzy Sync (<~>) --- // Initial supply: 500.00 // Initial demand: 600.00 // After fuzzy sync: // Market supply: 500.00 // Market demand: 500.00 // // --- Changing synced values --- // Supply changed to 700 // Market supply: 700.00 // Market demand: 700.00

Conditional Pull Arrow (<-[]-)

Pull a value only when a condition is met.

// COMPLETE EXAMPLE: Conditional Pull Arrow // Save as: conditional_pull.ouro "=== Conditional Pull Arrow ===" -> Print "--- Solar Panel Example ---" -> Print is_daytime := true solar_reading := 500.0 solar_output := 0.0 "Daytime: " + is_daytime -> Print "Solar reading: " + solar_reading -> Print // Conditional pull: only update if condition is true solar_output <-[is_daytime]- solar_reading "Solar output (daytime): " + solar_output -> Print "" -> Print "--- Night Time ---" -> Print is_daytime = false "Daytime: " + is_daytime -> Print // Reset and try again night_output := 999.0 // This pull should NOT happen because is_daytime is false night_output <-[is_daytime]- solar_reading "Night output (unchanged): " + night_output -> Print // Pull zero when NOT daytime night_output <-[!is_daytime]- 0.0 "Night output (zeroed): " + night_output -> Print // Expected Output: // === Conditional Pull Arrow === // --- Solar Panel Example --- // Daytime: true // Solar reading: 500.00 // Solar output (daytime): 500.00 // // --- Night Time --- // Daytime: false // Night output (unchanged): 999.00 // Night output (zeroed): 0.00

Conditional Pull with Transform

Transform the value while pulling conditionally.

// COMPLETE EXAMPLE: Conditional Pull with Transform // Save as: conditional_pull_transform.ouro "=== Conditional Pull with Transform ===" -> Print "--- Efficiency Adjustment ---" -> Print efficiency := 0.8 raw_reading := 100.0 is_active := true adjusted := 0.0 "Raw reading: " + raw_reading -> Print "Efficiency: " + efficiency -> Print "System active: " + is_active -> Print // Syntax: target <-[condition ? transformation]- source // Pull AND transform in one operation adjusted <-[is_active ? raw_reading * efficiency]- raw_reading "Adjusted reading: " + adjusted -> Print "" -> Print "--- Higher Efficiency ---" -> Print efficiency = 0.95 adjusted2 := 0.0 adjusted2 <-[is_active ? raw_reading * efficiency]- raw_reading "Adjusted with 95% efficiency: " + adjusted2 -> Print "" -> Print "--- System Inactive ---" -> Print is_active = false inactive_result := 999.0 inactive_result <-[is_active ? raw_reading * efficiency]- raw_reading "Result when inactive (unchanged): " + inactive_result -> Print // Expected Output: // === Conditional Pull with Transform === // --- Efficiency Adjustment --- // Raw reading: 100.00 // Efficiency: 0.80 // System active: true // Adjusted reading: 80.00 // // --- Higher Efficiency --- // Adjusted with 95% efficiency: 95.00 // // --- System Inactive --- // Result when inactive (unchanged): 999.00

Conditional Sync Arrow (<-[]->)

Synchronize values only when a condition is met.

// COMPLETE EXAMPLE: Conditional Sync Arrow // Save as: conditional_sync.ouro "=== Conditional Sync Arrow ===" -> Print "--- Secure Communication Sync ---" -> Print local_log := "Status: Online" remote_hq := "Standby" is_secure := true "Local log: " + local_log -> Print "Remote HQ: " + remote_hq -> Print "Connection secure: " + is_secure -> Print "" -> Print "--- Establishing Secure Sync ---" -> Print // Conditional sync: only sync if secure connection remote_hq <-[is_secure]-> local_log "Remote HQ (after sync): " + remote_hq -> Print "" -> Print "--- Updating Local Log ---" -> Print local_log = "Status: Mission Active" "Local log updated: " + local_log -> Print "Remote HQ (synced): " + remote_hq -> Print "" -> Print "--- Connection Compromised ---" -> Print is_secure = false "Connection secure: " + is_secure -> Print local_log = "Status: CRITICAL - Code Red" "Local log: " + local_log -> Print "Remote HQ (NOT synced - insecure): " + remote_hq -> Print // Expected Output: // === Conditional Sync Arrow === // --- Secure Communication Sync --- // Local log: Status: Online // Remote HQ: Standby // Connection secure: true // // --- Establishing Secure Sync --- // Remote HQ (after sync): Status: Online // // --- Updating Local Log --- // Local log updated: Status: Mission Active // Remote HQ (synced): Status: Mission Active // // --- Connection Compromised --- // Connection secure: false // Local log: Status: CRITICAL - Code Red // Remote HQ (NOT synced - insecure): Status: Mission Active

" + use_battery -> Print power <-[use_battery]- battery active_source <-[use_battery]- "Battery" // Priority 3: Grid (last resort) use_grid := !use_solar & !use_battery "Grid fallback? " + use_grid -> Print power <-[use_grid]- grid active_source <-[use_grid]- "Grid" "" -> Print "--- Result ---" -> Print "Active source: " + active_source -> Print "Power level: " + power -> Print // Expected Output: // === Priority Cascade Pattern === // --- Power Source Priority System --- // Solar available: 500.00 // Battery available: 300.00 // Grid available: 1000.00 // Power demand: 450.00 // // --- Evaluating Priorities --- // Solar sufficient? true // Battery fallback? false // Grid fallback? false // // --- Result --- // Active source: Solar // Power level: 500.00
