What is Ouroboros?

A New Paradigm

Ouroboros is a flow-oriented programming language designed from the ground up to make data movement explicit and visual. Instead of hiding how values travel through your program, arrows show the exact path data takes‚Äîmaking code easier to read, debug, and reason about.

Built for Clarity

Traditional languages obscure data flow behind assignments and return values. Ouroboros uses directional arrows (->, <-, <->) to make every data transfer visible at a glance. You'll always know where your data came from and where it's going.

Native Performance

Powered by the REX Engine (Rhombille Execution), Ouroboros compiles to optimized bytecode with SIMD acceleration. Whether you're building games, simulations, or tools, your code runs fast without sacrificing the language's expressive syntax.

Integrated Windowing

Create native windows, draw graphics, and handle input‚Äîall with built-in functions. No external libraries required. Build interactive applications, visualizations, and games directly in Ouroboros.

The Arrow System

Forward Arrow ->

Push data to a destination. Send values to functions, print to console, or chain operations together in a clear left-to-right flow.

"Hello, World!" -> Print value -> Transform -> Process -> Output

Pull Arrow <-

Capture results into variables. Also creates reactive bindings‚Äîwhen source values change, the bound variable updates automatically.

result <- Calculate(x, y) area <- width * height // Reactive!

Sync Arrow <->

Bidirectional binding keeps two values perfectly synchronized. Change either side and the other updates instantly‚Äîideal for UI state.

player_health <-> health_display // Either change reflects on both

Fork Arrow -<

Broadcast a single value to multiple destinations simultaneously. Perfect for event systems, logging, and parallel processing.

event -< Logger, Analytics, UI

Language Highlights

Flows, Not Functions

In Ouroboros, reusable code blocks are called Flows. They can return multiple values without tuples, accept any number of parameters, and chain naturally with arrows.

Flow Stats(a, b, c) sum := a + b + c avg := sum / 3 return sum, avg total, average <- Stats(4, 6, 10)

Domain System

Organize data with an Entity-Component architecture. Define reusable data structures in .var files, behaviors in .comp files, and compose them into entities with .ent files.

// Player.ent @use Position, Health, Inventory name := "Hero" max_health := 100

Constants with =:

Declare immutable values using the constant assignment operator. These values are set once and cannot be changed‚Äîperfect for configuration and magic numbers.

MAX_PLAYERS =: 4 PI =: 3.14159 WINDOW_TITLE =: "My Game"

Clean Control Flow

Familiar if/else, while, and for loops with clean syntax. Use & for AND, | for OR, and compound operators like +=.

for i := 0, i < 10, i += 1 if i > 5 & IsEven(i) i -> Print

Built-in Graphics

Create windows, draw shapes, render text, and handle mouse/keyboard input with native functions. Double buffering and anchor-based layouts included.

win <- WindowCreate("App", 800, 600) WindowDrawRect(win, 10, 10, 100, 50, 0x3498db) WindowDrawText(win, "Hello!", 20, 25, 0xFFFFFF)

Async Operations

Fire-and-forget with ->> or await results with <<-. Run tasks in background threads without callback complexity.

"background task" ->> Worker result <<- LongComputation(data)
