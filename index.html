Flow-Oriented Programming

Data flows. Code follows.

Ouroboros is a modern programming language where arrows define data flow. Forward, pull, sync, fork—every operation is a directed stream of values between expressions and flows.

Get Started See Examples Download

Why Ouroboros?

  • Directional Data Flow — Arrows show exactly where data goes
  • Multi-Return Flows — Return multiple values without tuples
  • Reactive Bindings — Variables update automatically when dependencies change
  • Bidirectional Sync — Keep two variables perfectly in sync
  • Conditional Gates — Route data based on conditions inline
  • Fork Broadcasting — Send data to multiple destinations at once

Core Concepts

Forward Arrow (->)

Push data forward to flows or print output.

"Hello, World!" -> Print 5 -> AddTwo -> AddTwo -> Print

Pull Arrow (<-)

Capture return values or create reactive bindings.

result <- Add(10, 5) // Reactive: area updates when width/height change area <- width * height

Sync Arrow (<->)

Bidirectional binding—changes on either side mirror to the other.

player_hp <-> ui_hp_display player_hp = 75 // ui_hp_display now 75

Fork Arrow (-<)

Broadcast a value to multiple destinations simultaneously.

event_data -< LogEvent, UpdateUI, PlaySound

Quick Examples

Define a Flow

Flow Add(a, b) result := a + b return result sum := Add(10, 5) sum -> Print // Outputs: 15.00

Multi-Return

Flow Stats(a, b, c) sum := a + b + c avg := sum / 3 return sum, avg total, average <- Stats(4, 6, 10) total -> Print average -> Print

Conditional Gates

// Only execute if condition is true value -[value > 50]-> ProcessHigh // Transform data inline input -[input > 0 ? input * 2]-> Process

Control Flow

if score ≥ 90 grade := "A" elseif score ≥ 80 grade := "B" else grade := "C" while counter < 10 counter += 1

Advanced Features

Continuous Funnels (▷-)

Stream data reactively—any change triggers the target.

sensor_temp ▷- ThermalMonitor sensor_pressure ▷- ThermalMonitor

Fuzzy Logic (~>)

Soft influence—suggest rather than set values.

temperature ~> ClimateController

Solvers (=>)

Constraint-based pathfinding or optimization.

start_position => PathSolver

Ownership Transfer (-|->)

Move ownership of data—source becomes invalid.

cargo -|-> ReceiveShipment
